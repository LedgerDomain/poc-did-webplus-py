# did:webplus Test Vectors

Canonical test vectors for did:webplus interoperability testing. Generated by `cargo run -p did-webplus-test-vectors`.

## Format

- **JCS**: JSON Canonicalization Scheme (RFC 8785) - use `serde_json_canonicalizer` for compatibility
- **Hash function**: SHA3-256 with Base64Url encoding (multicodec)
- **JWS proofs**: Detached payload, `b64: false`, `kid` = base64url multicodec public key
- **Update rules `key`**: base64url (recommended) or base58btc supported; `key-base58btc.json` demonstrates did:key interoperability

## Directory Structure

- `valid/` - Valid root and non-root DID documents
- `microledgers/` - Complete did-documents.jsonl files
- `did-to-url-mapping/` - DID string to resolution URL mapping
- `negative/` - Invalid documents (rejection expected)
- `resolution-expectations/` - Expected resolution output
- `update-rules-variants/` - Update rules examples (key, key-base58btc, hashedKey, any, all, atLeast)

## Usage for Implementors

1. Parse DID documents from JCS strings
2. Verify selfHash matches computed hash
3. For non-root documents, verify chain (prevDIDDocumentSelfHash, versionId, validFrom)
4. Verify JWS proofs and update rules
5. See did:webplus spec: https://ledgerdomain.github.io/did-webplus-spec
